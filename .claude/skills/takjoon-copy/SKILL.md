# 백준/택준 제출 형식으로 클립보드 복사

택준(takjoon.takijk.xyz) 제출 형식은 **백준(BOJ)과 동일**하다: `package` 없이
`public class Main`, stdin으로 입력을 읽고 stdout으로 정답만 출력한다.

대상 파일: $ARGUMENTS (없으면 대화에서 현재 풀고 있는 풀이 파일을 사용)

## 프로세스

1. **풀이 파일 읽기**: 대상 `.java` 파일을 Read한다. (인자 없으면 직전까지
   작업한 파일. 불명확하면 사용자에게 어떤 파일인지 확인한다.)

2. **입력 형식 파악**: 변환의 핵심. 풀이 메서드의 시그니처와 `main`의 예제
   테스트 케이스를 보고 stdin 파싱 코드를 만든다.
   - `@link`의 문제 페이지/예제 입력으로 입력 줄 구성을 추론한다.
   - 예: `int[][] questions` + 예제 `{{123,1,1},...}` → 첫 줄 N, 이후 N줄에
     공백 구분 정수 3개.
   - 추론이 애매하면(입력 줄 순서·개수·구분자) **추측하지 말고 사용자에게
     예제 입력 형식을 묻는다.**

3. **Main 클래스로 변환**:
   - `package ...;` 선언 제거.
   - 클래스명을 `Main`으로 변경.
   - 기존 풀이 메서드(`solution` 등)와 헬퍼 메서드는 `static`으로 바꿔 보존.
     로직은 **절대 수정하지 않는다**.
   - `main`을 stdin 파싱 → 풀이 메서드 호출 → `System.out.println(결과)` 형태로
     교체. 입력 파싱은 `BufferedReader` + `StringTokenizer` 사용.
   - 출력은 정답 값 하나만(예제 라벨·디버그 출력 없이).

4. **클립보드 복사**: 변환 결과를 `pbcopy`로 복사한다. 헤어독으로 전달:
   ```bash
   pbcopy <<'EOF'
   <변환된 코드>
   EOF
   echo "복사 완료 ($(pbpaste | wc -l) 줄)"
   ```

5. **결과 통지**: 복사됐음을 짧게 알리고, 변환된 코드를 코드블록으로 보여준다.

## 규칙

- **택준 제출 형식 = 백준(BOJ)과 100% 동일하다.** `package` 없이
  `public class Main`, stdin으로 입력을 읽고 stdout으로 정답만 출력. 프로그래머스
  스타일(`class Solution` + `solution` 메서드 반환)이 아니다. 이 형식이
  변환의 고정 목표다.
- 풀이 로직(알고리즘·헬퍼 메서드 내용)은 한 글자도 바꾸지 않는다. 껍데기
  변환(package/클래스명/입출력)만 한다.
- **코드 정리·리팩토링·포매팅을 하지 않는다.** 사용자가 작성한 코드를 그대로
  두고, 제출 형식(class Main + stdin/stdout)에 맞게 껍데기만 조정한다. 변수
  합치기, 줄바꿈 정돈, 네이밍 변경 등 미관 개선을 임의로 추가하지 말 것.
- 파일에 디스크 변경을 가하지 않는다. 클립보드로만 내보낸다.
- 입력 형식이 불확실하면 추측하지 말고 사용자에게 확인한다.
- macOS 전용(`pbcopy`/`pbpaste`).
